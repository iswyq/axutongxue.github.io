关于这个博客

原本的目的

搭建本站最起初是为了备份自己微信公众号的文章

顺便借助本站实现微信一直不重新开通的留言功能

接近一年的时间里发布的文章，我都在这个博客上进行了文章备份。

暂时弃用

由于最起初的想法和计划不周，同时出于以下原因，决定弃用本站（以后不再在本站上更新）

更新发布繁琐

Hexo+Github搭建的此博客，想要生成文章、修改文章都比较麻烦，需要先在本地渲染好之后，再push到github来发布

而我分享的各种文章又需要经常更新，如此一来上面的操作就显得非常麻烦！

评论系统问题

本站采用的Valine评论系统的邮件通知功能问题很大，总是不提示有新评论，导致我也没办法及时回复

而且随着公众号的规模增大，一个人运营全部的我，面对如此多的消息需要回复，我暂时分身乏术：

  • 公众号后台消息
  • 储物间留言
  • 知乎私信
  • 3个QQ群的消息
  • 需要写新文章

于是心想着既然无法提供更好的服务，就干脆关掉它

本站之后将会删除评论功能

新的备份方式

几经思索，最后，选择换用更简单方便的github issue来实现文章备份+评论+资源分享：https://github.com/axutongxue/Article-backup/issues

即我会在github issue的文章中直接给出资源连接

好处

github issue有几个好处

  • 通过github issue的tag 和 project 可以轻松的实现标签、归档等博客一样的功能
  • 有搜索功能
  • 自带评论功能，同时因为需要github帐号才能评论，无形中屏蔽了垃圾评论
  • 无需本地渲染，打开网页即可在线编辑文章！
  • 访问速度比github page更快
  • 自带图床！于是不必担心图片失效的问题
  • 不对搜索引擎开放也正契合我的需求：想要知道我的github issue地址，必然是关注了储物间，或者公众号（阿虚同学）的粉丝

其实折腾来折腾去，也只是想提供更好的服务
